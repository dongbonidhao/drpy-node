# 多引擎源（php/hipy/cat）修复报告

> 时间: 2026-07-18（第三轮）
> 范围: report.json 失效源里的 php(10)/hipy(38)/cat(7) + ds 残余(8)，以及 ds 之外三种引擎的修复与 skill 文档完善
> 工具: 直连 drpy-node 服务 API（`?do=py|php|cat&extend=`）+ node 诊断脚本

## 一、重大发现：report.json 裸测误判

`report.json` 生成脚本用裸 `?do=py&pwd=xxx` 测试，**不带 extend 参数**。而 hipy/php 的模板源（AppFox/AppGet/AppSk/getapp3.4.4 等被多站共用的模板）依赖 ext 配置 host/key，裸测必报 `Invalid URL`/`KeyError 'host'`，导致整批模板源被误判失效。

**正确测试方法**：先拉订阅 `http://localhost:5757/config/1?sub=all` 取每个源的 ext，再带 `&extend=<ext>` 重测。ext 是字符串（URL）或 JSON 对象（{host,key}），传参时 JSON.stringify + encodeURIComponent。

带 extend 重测后，原本 116 个"失效"源里大量其实正常（class 非空）。

## 二、hipy 修复（核心成果）

### 根因
hipy 走 `base/spider.py` 基类，`fetch`/`post` 默认 `verify=True`，且 requests `trust_env` 默认读 **Windows 系统代理**（非环境变量）。两类问题：
1. **SSL 证书验证失败**：证书有问题的站点（app.omofun1.top / dy.58ys.vip / 1.star2.cn 等）`SSLCertVerificationError`
2. **系统代理超时**：http host 走 Windows 系统代理 127.0.0.1:7890，`HTTPConnectionPool Read timed out`

### 修复
`spider/py/base/spider.py` 的 `fetch`/`post`/`postJson`/`postBinary` 默认 `verify=False` 并传 `proxies={}`。框架级合理默认（爬虫直连、不验证证书），对正常源无副作用。`spider/py/剧透社[盘].py` 自建 `requests.Session`，补 `session.verify=False` + `session.get(verify=False)` 双保险。

### 效果：新修 7 个 SSL/代理问题源

|---|---|---|

| 魔方[AG¹] | snysw.xyz 取 host SSL | base verify=False |
| 哇哇APP | SSL | base verify=False |

另 7 个 hipy 源（爱玩音乐/爱听音乐/河马短剧/兄弟影视/瓜子/新韩剧网/资源管理）重测已正常（report 快照偶发失效，本就可用）。

### hipy 改源关键坑
hipy 走 `t4_daemon.py` 常驻 python 进程，**缓存已 import 的 module**。改 `base/spider.py` 或任何 `.py` 后必须 kill daemon 重载（ds/cat 改了直接生效无此问题）：
```bash
# Windows kill
powershell -Command "Get-CimInstance Win32_Process -Filter \"name='python.exe'\" | Where-Object {\$_.CommandLine -like '*t4_daemon*'} | ForEach-Object { Stop-Process -Id \$_.ProcessId -Force }"
# 重启 drpy-node 服务自动 spawn 新 daemon，或手动:
python spider/py/core/t4_daemon.py --pid-file t4_daemon.pid --log-file logs/daemon.log --host 127.0.0.1 --port 57570
```
