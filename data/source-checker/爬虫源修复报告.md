## 附录:方法说明

- **存活检测脚本**:`_fix_work/alive_check.cjs`(node 并发 HTTP)、Playwright 复检存疑域名
- **补提取网址**:`_fix_work/resolve_no_url.cjs`(用 cli resolved 提取 DS 加密源的 host)
- **修复**:主进程(Playwright 桌面端分析)+ 3 个并行 subagent(cli 移动端 fetch 分析+迭代 test)
- **验证**:`_fix_work/test_changed.cjs`、`test_unfixed.cjs`(cli test home,要求 success && item>0)
- **中间产物**:`_fix_work/` 下 error_sources.json、alive_check.json、no_url_resolved.json、test_changed.json 等

### 已知限制
1. cli 测试引擎使用**移动端 UA** 抓取,与桌面端结构可能不同,修复以 cli fetch 到的移动端结构为准
2. 依赖 `_lib.request` 等模块的源(七味等)在 cli 测试沙箱中无法初始化,需在真实 drpy-node 服务实测
3. 部分源"推荐列表空"属推荐区块选择器失效,分类可用,建议后续单独调试推荐选择器
