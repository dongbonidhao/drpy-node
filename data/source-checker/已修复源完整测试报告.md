## 一、本轮关键成果

### 1. 樱花动漫[优] 修复确认（dict 二级正确写法）

- **根因**：dict 二级 `tabs` 误带 `&&Text`（`tabs:'.play-list-group-switch-item&&Text'`），jsoup 把整串当 css 选择器返回空 -> playFrom=[] -> lists 循环 0 次 -> 播放线路全空。**不是 `#id` 机制问题**（`#id` 替换逻辑在 `libs/drpysParser.js` line 1261，可靠）。
- **规律纠正**：tabs 能带 `&&`（`&&` 在 pdfa 里当后代选择器，等价空格），但 `&&` 后必须是元素选择器(标签 `a`/`dt`/`li`、`.class`、`#id`)，不能是 `Text`/`Html`/`href` 等 pdfh 取值段。模板 `libs_drpy/template.js` 就有 `.play_source_tab&&a` 合法写法。
- **修复**：`tabs` 改 `'.play-list-group-switch-item'`（取 3 个自身元素当 tab）。用有线路的影片(如 id=4425)测，`vod_play_from=1-30$$$31-60$$$61-62`，count=60，play 通过，**规则正确**。
- **注意**：evaluate 取片会波动--若串联取到无线路的片(如首页首项 id=3856，该详情页 `.play-list-group-switch-item` pdfa count=0)会显示 count=0 误判不可用。属 evaluate 取片策略局限，非源规则问题。手动 `test detail --ids <有线路影片URL>` 可确认规则本身正确。

### 2. drpy dict 二级认知纠正（重要）

此前误判"drpy 引擎 dict 二级 `#id` 机制不可靠、需改 async"，**错误**。源码核实（`libs/drpysParser.js` `commonDetailListParse`）：
- `#id`/`#idv` 替换完全可靠（line 1261）
- lists 循环次数 = tabs 解析出的元素数（playFrom 长度）
- **真正高频陷阱**：`tabs` 是 pdfa 选择器，不能带 `&&`。`&&` 后是 pdfh 残片(`Text`/`href`)时 pdfa 返回空；是合法标签(`dt`)时返回偏离语义元素。两种都错。
